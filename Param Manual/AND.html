PARAM Utilities AND, ONLY, NOT and ALL

  These four utilities use the program SELECT to build a list of data sets that match certain criteria.  They are all used at the DOS prompt, i.e., in a console window.  They should only be used in a folder that contains a set of MOPAC data files.  Before any of them are used, the utility FILES should have been run.  FILES will create a file called "files.out" that the utilities use.  Each of the utilities selects data files according to different criteria.  These criteria are:

AND:  Selects all compounds that contain one or more of a user-specified set of elements. Other elements can also be present.  Thus if AND was used with the element Ti, then all species in the database that contain one or more atoms of titanium would be selected.

ONLY: Selects all compounds that contain one or more of a user-specified set of elements. All other elements are excluded.  Thus if ONLY was used with the elements H, C, N, and O,  then all species in the database that contain one or more atoms of H, C, N, or O would be selected.  This would include H2, CH2O, C6H5NO2, etc., but not CH3F.

NOT: Selects all compounds that do not contain any of the specified elements.

ALL: Selects all compounds.

Procedure for running these utilities

They can only be run in a console window.  At the DOS prompt, enter the appropriate command, followed by a return. The utility will then write out the line: "Enter chemical symbols for search"

Type in all the elements to be selected.  Elements are represented by their symbols, each element is separated from the next one by a space.  Thus if the elements hydrogen, carbon, oxygen and nitrogen are wanted, type in "H C N O".  If all elements are wanted, use the unique symbol "all"  Of course, do not use 'all' with the NOT utility.

Take a return.  The utility will then print a message of the type: "NO. OF HEATS: 4546, DIPOLES: 569, I.P.s: 670, GEOREFS:3655"  This shows how many data of each type were selected.

The utility will then generate a file called "bits".  "bits" is a list of data files that can be used by PARAM. As soon as "bits" is created, a useful operation would be to copy or move it to the folder where PARAM is to be run from.
